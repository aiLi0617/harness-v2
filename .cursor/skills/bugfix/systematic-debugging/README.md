## 这个 Skill 是做什么的

`systematic-debugging`（结构化调试流程（Systematic Debugging））定义了标准化执行流程。

本技能定义了从 Bug 报告到根因确认的完整调试流程。所有调试活动必须按步骤执行，严禁跳步。

- 主要产出：**`docs/artifacts/root-cause.md`**
- Agent 执行细节以 [`SKILL.md`](./SKILL.md) 为准

### 适用场景

- 收到 Bug 报告需要复现和定位
- 生产环境异常需要根因分析
- 测试失败需要排查原因

### 前置条件

- 已获取 Bug 描述（至少包含：现象、期望行为、实际行为）
- 有权访问相关日志和代码仓库
- 本地开发环境可运行项目

---

## 流程概览

整体为 **5 步**流程（由 SKILL.md 同步）；禁止跳步。

```
Bug 报告 / 异常信息
        ↓
┌───────────────────┐
│ 1. 收集信息 │  建立对 Bug 的全面认知，避免信息缺失导致方向偏差。
└─────────┬─────────┘
          ↓
┌───────────────────┐
│ 2. 形成假设列表 │  基于收集到的信息，列出至少 3 个可能的根因假设，按可能性排序。
└─────────┬─────────┘
          ↓
┌───────────────────┐
│ 3. 验证假设 │  逐个验证假设，用证据排除或确认每个假设。
└─────────┬─────────┘
          ↓
┌───────────────────┐
│ 4. 确认根因 │  排除其他可能性，锁定具体代码位置和触发条件。
└─────────┬─────────┘
          ↓
┌───────────────────┐
│ 5. 输出根因分析报告 │  将分析过程和结论输出为结构化文档，供后续修复和回顾使用。
   （下游）test-driven-bugfix → 写失败测试 → 最小修复
```

### 各步骤要点（摘要）

| 步骤 | 名称 | 目标 |
|------|------|------|
| 1 | 收集信息 | 建立对 Bug 的全面认知，避免信息缺失导致方向偏差。 |
| 2 | 形成假设列表 | 基于收集到的信息，列出至少 3 个可能的根因假设，按可能性排序。 |
| 3 | 验证假设 | 逐个验证假设，用证据排除或确认每个假设。 |
| 4 | 确认根因 | 排除其他可能性，锁定具体代码位置和触发条件。 |
| 5 | 输出根因分析报告 | 将分析过程和结论输出为结构化文档，供后续修复和回顾使用。 |

### 与工作流阶段的对应关系

在 `bugfix.yaml` 中，本 Skill 覆盖两个阶段：

| 工作流步骤 | 本 Skill 承担的内容 |
|-----------|-------------------|
| `reproduce` | 步骤 1：收集信息；本地复现并记录结果 |
| `root-cause-analysis` | 步骤 2–5；由 `bug-analyst` 代理执行并写出 `docs/artifacts/root-cause.md` |

### 严格禁止（摘要，详见 SKILL.md）

- 未分析就直接改代码
- 同时改多处碰运气
- 跳过假设验证直接下结论
- 忽略偶现 Bug 的复现条件分析
- 删除或注释掉出错代码了事
- 不记录分析过程

完整列表见 [`SKILL.md` 禁止事项](./SKILL.md#禁止事项)。

## 如何使用

### 方式一：走 Bug 修复工作流（推荐）

1. 在 Cursor 中描述 Bug（现象、期望/实际行为、日志或堆栈、复现步骤）。
2. 说明要按 **bugfix 工作流** 处理，或明确要求使用 `systematic-debugging`。
3. Agent 会按 `bugfix.yaml` 调度：先复现与收集信息，再调用 `bug-analyst` + 本 Skill 完成根因分析。
4. 确认 `docs/artifacts/root-cause.md` 已生成且包含：根因位置、触发条件、建议修复方向。
5. 再进入修复阶段（`test-driven-bugfix` + `implementer`），**不要**在根因报告缺失时直接改业务代码。

示例提示词：

```text
请按 bugfix 工作流处理这个 Bug，根因分析阶段使用 systematic-debugging，
产出 root-cause.md 后再进入修复。

【现象】下单接口返回 500
【期望】创建订单成功
【实际】NullPointerException
【堆栈】（粘贴完整堆栈）
【复现】POST /api/orders，body: {...}
```

### 方式二：仅做根因分析（不修复）

```text
请只读分析这个 Bug，读取 .cursor/skills/bugfix/systematic-debugging/SKILL.md 并严格执行，
由 bug-analyst 输出 docs/artifacts/root-cause.md，不要修改任何源代码。
```

### 方式三：人工按 Skill 自检

开发者也可对照 [`SKILL.md`](./SKILL.md) 自行排查：

1. 打开 SKILL.md，从步骤 1 开始逐项执行。
2. 用 SKILL 中的「假设列表」表格记录假设与验证结果。
3. 分析结束后，按 SKILL 末尾的报告模板填写 `docs/artifacts/root-cause.md`。
